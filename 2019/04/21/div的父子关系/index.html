div的父子关系

子div有内容，父div的高度为何为0？

<!DOCTYPE html>
<html>
<head>
    <title></title>
    <style>
        .a{    
            background-color: orange;
        }
        .b{
            height: 50px;
            width: 50px;
            background-color: blue;
            float: left;
        }
    </style>
</head>
<body>
    <div class="a">
        <div class="b"></div>
    </div>
</body>
</html>

效果图:

一个父视图包含一个子视图，子div有高度，父div高度为0

原因

当一个元素只包含浮动元素的时候，它会出现高度折叠,即元素的上下底边重合，和高度为0效果一样。

当子div用绝对定位时

.b{
    height: 50px;
    width: 50px;
    background-color: blue;
    position: absolute;
    left: 70%;
}

父div高度j依旧为0，因为子元素绝对定位脱离文档流，不会撑开父元素，父元素高度为0。

解决方案

  • overflow的hidden属性
    父div添加overflow的hidden属性
  • 添加一个空的div
    <div class="a">
    <div class="b">
    </div>
    <div style="clear: both;"></div>
</div>

这种方法是添加一个空的div，这个div和浮动元素同一级别，且位于浮动元素的最后。它在这里并不具有实际意义，它的作用只是撑开div标签而已。
